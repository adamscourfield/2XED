### Gap 1.1 — Two competing sequence models inside one skill batch
- Items 1–5 are modelled as `sequence_size: 5`.
- Then another run starts with `sequence_size: 25` and positions 1–15.
- This creates ambiguous progression interpretation for teachers/analytics.

**Edit action**
1. Split into explicit sequence groups by adding a sequence identifier in notes (or internal metadata convention), e.g.:
   - `N1.3-SEQ-A` (concept MCQ set, size 5)
   - `N1.3-SEQ-B` (comparison fluency set, size 15)
2. Keep position/size internally consistent per group.
3. If platform expects one progression only, flatten into one coherent 1..20 sequence with intentional difficulty gradient.

### Gap 1.2 — Difficulty gradient is shallow
- Current profile is heavily L2 with limited L3 transition (17/20 at L2).
- Few transfer-style prompts from symbolic to contextual reasoning.

**Edit action**
1. Add/upgrade 3–4 items to L3 via:
   - chained inequalities (e.g., infer ranges from two inequalities),
   - boundary comparisons in context,
   - reverse-statement translation with distractor rationale.
2. Place those items toward end of sequence to create visible progression.

## Priority 2 (Diagnostic quality / misconception usefulness)

### Gap 2.1 — Misconception tagging is over-concentrated
- `OVERGENERALISATION` is used for 90% of N1.3 items.
- Limits precision of remediation routing and post-hoc analysis.

**Edit action**
Diversify misconception labels where appropriate, e.g.:
- `PROCEDURAL_SLIP`: sign-direction reversal in translation tasks.
- `TRANSCRIPTION_ERROR`: copying symbol/number incorrectly.
- `FACT_RETRIEVAL_GAP`: uncertainty with language like "at least", "at most", "minimum".
- Keep `OVERGENERALISATION` only where it truly fits the wrong route.

### Gap 2.2 — Question-type mix is narrow for high-stakes checks
- 15/20 are `SHORT` statements; only 5 MCQ with options.
- Short-only in early foundation can hide misconception routes unless answer capture supports rich diagnostics.

**Edit action**
1. Convert 4–6 `SHORT` items into MCQ or TRUE/FALSE + justification capture.
2. Prioritize conversion for known confusion points:
   - strict vs non-strict (`>` vs `≥`),
   - equivalence translation,
   - boundary value inclusion.

---

## Priority 3 (Polish / consistency)

### Gap 3.1 — Placeholder and symbol readability
- Some stems/options use `?` placeholders and compressed symbolic phrasing.
- Risk: readability friction in student UI and content QA.

**Edit action**
1. Standardize placeholder variable style (`x`, `n`, etc.) unless source requires `?`.
2. Normalize verbal equivalents in stems (e.g., "at least", "no more than") to reduce ambiguity.
3. Keep symbol forms in options to preserve translation practice.
